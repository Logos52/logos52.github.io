# Norman Doors

A Norman Door is a door (or any interface) whose design is so poor that users cannot intuitively understand how to operate it. The user is then blamed for the failure ("Just push/pull harder"), when the real problem is the design itself.

Named after Don Norman and popularized in his book *The Design of Everyday Things*.

## Core Thesis

Bad design often manifests as **design gaslighting** — the system makes the user feel incompetent for failing to use something that was never properly designed in the first place. The fault lies with the designer, not the user.

## Key Mechanisms

**Poor Affordances**  
The object fails to clearly communicate how it should be used. A door with no visible push plate or handle gives no clue whether it should be pushed or pulled.

**Lack of Signifiers**  
There are no clear signals (labels, arrows, texture, shape) indicating the correct action.

**Blame the User Culture**  
When repeated failures occur, the response is often to add instructions ("Push") or to train users, rather than fixing the underlying design problem.

## Broader Application

The Norman Door concept extends far beyond physical doors:
- Confusing software interfaces
- Poorly designed websites and apps
- Bureaucratic processes that require "figuring out" the system
- Any situation where the user is expected to adapt to bad design instead of the design adapting to the user
