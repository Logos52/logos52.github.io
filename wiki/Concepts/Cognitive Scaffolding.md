# Cognitive Scaffolding

Cognitive scaffolding refers to design parameters and structures that ease the learning curve when acquiring a new skill, from beginner to competent practitioner.

The core idea is to temporarily reduce complexity so that a learner can focus on the most important elements first, then gradually remove the supports as competence grows.

## Core Thesis

Good scaffolding does not just teach content — it manages cognitive load by controlling what the learner pays attention to at each stage. Effective scaffolding makes early progress feel achievable while still building toward full capability.

## Key Mechanisms

**Narrative Scaffolding**  
Story and context can pull a learner in emotionally and give meaning to the skill before technical mastery is achieved. A compelling "why" often lowers the barrier more effectively than pure instruction.

**Constraint Scaffolding**  
Temporarily limiting the environment or interface so the learner only interacts with relevant parts. Examples:
- Training modes in software that blur out unrelated UI elements
- Simplified versions of a task that remove advanced variables
- Guided workflows that only present one decision at a time

**Progressive Disclosure**  
Revealing complexity gradually rather than all at once. The learner masters core loops before being exposed to edge cases or advanced features.

## Practical Applications

- Designing learning systems and interfaces
- Creating onboarding flows for complex tools
- Teaching complex skills (programming, languages, instruments, sports)
- Building personal practice routines (starting with highly constrained versions of a skill)

The goal is to lower the activation energy for early practice while preserving the path to real competence.
