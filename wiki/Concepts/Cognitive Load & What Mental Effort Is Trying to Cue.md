# Cognitive Load & What Mental Effort Is Trying to Cue

Cognitive load is not just friction. It is information about what the mind is trying to hold, resolve, compare, or transform.

## Summary

The transcripts frame mental effort as a metacognitive doorway. Too little load can mean passive processing. Too much load can mean working memory overload. The practical skill is reading the signal correctly and responding with the right move.

[[wiki/Syntheses/First Principles of ICS|First Principles of ICS]] adds the most useful practical version: overwhelm is the cue. The learner should make the overwhelm explicit, turn it into a question, and use the question to organize the next move.

## Signal Types

| Signal | Possible Meaning | Useful Response |
| --- | --- | --- |
| Low effort, drowsiness, drifting | Passive processing | Increase active transformation: teach, map, question, compare |
| High effort with progress | Productive deep processing | Continue, but monitor fatigue |
| High effort with confusion | Working memory overload | Pause, narrow scope, externalize, separate concerns |
| Overwhelm | Triage problem | Identify the biggest constraint |
| Repeated stuckness | Strategy mismatch | Change angle or practice condition |

## Working Memory Constraint

Working memory is limited. Trying to resolve many interacting factors internally can create confusion. Useful responses include:

- Separating concerns.
- Dependency mapping.
- Thinking on paper.
- Pausing consumption.
- Handling information immediately.
- Narrowing the question.

## Metacognitive Use

The key question is not "Is this hard?" but "What kind of hard is this?"

- If it is hard because the learner is making meaningful comparisons, the difficulty may be productive.
- If it is hard because too much is being held at once, the next move is offloading or decomposition.
- If it is easy because the learner is coasting, the next move is to raise processing depth.
