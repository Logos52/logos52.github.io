# Exit Strategy For Objects

Exit Strategy For Objects is the rule for how an object will leave before it enters.

Every purchase creates a future removal problem. A clean exit strategy reduces hesitation, resale delay, storage buildup, and guilt.

## Exit Options

Common exits:

- use up;
- return;
- sell;
- give away;
- recycle;
- discard;
- rent instead of buy;
- borrow or share;
- use a city service instead of owning.

The best exit is the one that removes the object with the least unnecessary friction while preserving the values that matter.

## Selling Is A Tool

Selling works best for objects with real demand, clear value, and manageable logistics.

Selling becomes a trap when the object sits for weeks because the owner wants to recover a small amount of money.

Use this rule:

```text
sell when demand is real and removal stays fast
give away or discard when resale friction preserves clutter
```

## Trash Can Be The Clean Exit

Discarding can be the correct exit when the cost has already been paid and resale creates more work than value.

The lesson should still be captured:

```text
why did this object fail?
what buying rule changes now?
```

The object leaves. The lesson remains.

## Design Purchases With Exits

Before buying, ask:

1. How will this leave?
2. Will I use it up?
3. Can it be resold easily?
4. Can I rent or borrow instead?
5. Will this create storage or maintenance debt?
6. What rule tells me when to remove it?

This turns shopping into decision design.

## Relationship To Ownership Cost

[[wiki/Minimalism/Ownership Cost|Ownership Cost]] includes exit cost.

Large, awkward, low-demand objects have high exit cost. A cheap purchase can become expensive if it is difficult to remove.

## Practical Use

For objects already owned:

1. Choose the cleanest exit.
2. Set a short deadline.
3. Avoid maximizing resale when speed matters more.
4. Reflect on why the object failed.

For future objects:

1. Define the exit before buying.
2. Prefer rentals or shared services for rare-use items.
3. Avoid purchases with unclear removal paths.
