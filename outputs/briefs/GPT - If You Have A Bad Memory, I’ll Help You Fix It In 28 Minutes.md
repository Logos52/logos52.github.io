# GPT - If You Have A Bad Memory, I'll Help You Fix It In 28 Minutes

## Core Thesis

The source reframes "bad memory" as usually a method problem. People often blame their memory when the real issue is weak encoding, shallow attention, poor organization, lack of retrieval, or mismatched review.

Memory improves when information is made meaningful, connected, retrievable, and reviewed under the right conditions.

## Compressed Takeaways

1. **Bad memory is often bad encoding.** If information was never processed deeply, recall will be weak.
2. **Attention matters.** The brain cannot remember what it did not properly register.
3. **Organization drives retrieval.** Scattered notes produce scattered recall.
4. **Retrieval must be practiced.** Recognition is not the same as recall.
5. **Memory is context-sensitive.** Information should be encoded near the way it will be used.
6. **Review should repair gaps.** Repetition alone is not enough.

## Practical Memory Workflow

1. Clarify what the information is for.
2. Connect it to prior knowledge.
3. Organize it into a structure.
4. Retrieve it without looking.
5. Identify what was missing.
6. Repair the map.
7. Space and interleave future retrieval.

This is the combined logic of [[wiki/Techniques/Bear Hunter System|Bear Hunter System]] and [[wiki/Techniques/Spaced Interleaved Retrieval|Spaced Interleaved Retrieval]].

## What This Should Feel Like

It should feel like building a memory system rather than testing whether you are naturally good at remembering.

It should not feel like rereading, highlighting, and hoping familiarity becomes recall.
