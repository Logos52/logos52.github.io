# GPT - How to Learn Anything Faster Using Modern Research

## Core Thesis

The source frames fast learning around a small set of durable learning rules rather than hacks. Learning improves when the learner uses active processing, spaced retrieval, meaningful organization, feedback, and metacognitive adjustment.

## Compressed Takeaways

1. **Learning is not exposure.** Time with material is not the same as encoding.
2. **Active processing matters.** The learner needs to transform information.
3. **Retrieval strengthens memory.** Recall should be part of the learning process.
4. **Spacing improves durability.** Repetition works best when timed and effortful.
5. **Interleaving improves discrimination.** Mixing related material strengthens flexible use.
6. **Metacognition keeps the system adaptive.** Learners need to notice when methods stop working.

## Practical Workflow

1. Preview the topic and identify main structures.
2. Build an initial map.
3. Learn in chunks.
4. Retrieve before rereading.
5. Space reviews.
6. Interleave similar concepts.
7. Use mistakes to update the map.

This is basically the public-facing logic behind [[wiki/Syntheses/ICS System|ICS System]].

## What This Should Feel Like

It should feel effortful but clarifying. The learner should often be forced to recall, compare, and reorganize.

It should not feel like smooth passive familiarity.
