# GPT - Learn to Learn in 4hrs 54mins - Full Course

## Core Thesis

This full course presents learning as an integrated system built from retrieval, encoding, mind mapping, and skill acquisition. The main idea is that efficient learning is not one technique. It is the coordination of multiple processes: how information is first processed, how it is structured, how it is retrieved, how it is corrected, and how it becomes usable in performance.

The course is the broad public backbone for many of the user's ICS-related pages.

## Compressed Takeaways

1. **Retrieval is not just review.** It is a tool for diagnosis, strengthening, and gap repair.
2. **Encoding determines retrieval quality.** Poorly structured information is hard to recall and apply.
3. **Mind mapping is a thinking method.** The map should represent relationships, not decorate notes.
4. **Learning happens in layers.** Start with structure, then refine detail.
5. **Skills require feedback loops.** Performance improves through action, reflection, and adjustment.
6. **Metacognition coordinates the system.** The learner must monitor what is working and why.
7. **Passive learning habits must be unlearned.** Rereading and linear copying feel easy but produce weak mastery.
8. **The system should become self-regulating.** The learner should know which process is failing.

## Retrieval

The retrieval sections emphasize that recall is not only for checking memory after learning. It is part of learning itself.

Retrieval should:

- expose gaps,
- strengthen access,
- improve discrimination,
- force reconstruction,
- guide what to review next.

This aligns with [[wiki/Techniques/Spaced Interleaved Retrieval|Spaced Interleaved Retrieval]]. The point is not to make review more painful. The point is to make review more informative.

## Encoding

The encoding sections emphasize that new information needs to be processed into a useful structure. If encoding is shallow, retrieval becomes unreliable.

Good encoding includes:

- identifying importance,
- asking useful questions,
- connecting to prior knowledge,
- organizing relationships,
- simplifying without losing meaning,
- preparing information for future use.

This maps directly to [[wiki/Techniques/Bear Hunter System|Bear Hunter System]] and its Aim, Shoot, Skin sequence.

## Mind Mapping

The mind mapping sections distinguish real mapping from decorative diagrams. A good map is not just a radial outline. It is a representation of relationships.

Useful maps:

- show hierarchy where hierarchy exists,
- show lateral relationships,
- keep groups manageable,
- avoid clutter,
- change as understanding changes,
- support later reconstruction.

This connects to [[wiki/Concepts/Importance-Based Chunking|Importance-Based Chunking]] and [[wiki/Concepts/Deep Processing Practice|Deep Processing Practice]].

## Skills Acquisition

The skills sections move from knowledge toward performance. Skill learning requires doing, feedback, adjustment, and repeated calibration.

The practical loop:

1. Attempt the skill.
2. Observe the result.
3. Diagnose the bottleneck.
4. Adjust the method.
5. Practice under better constraints.
6. Reflect and repeat.

This connects to [[wiki/Techniques/Kolbs Experiential Cycle|Kolbs Experiential Cycle]] and [[wiki/Techniques/Marginal Gains|Marginal Gains]].

## System View

The course is most useful when treated as a system:

- Encoding builds the structure.
- Retrieval tests and strengthens the structure.
- Mind mapping externalizes and refines the structure.
- Skill practice turns knowledge into action.
- Metacognition monitors the whole process.

This is essentially the logic of [[wiki/Syntheses/ICS System|ICS System]].

## What This Should Feel Like

It should feel like you are managing a learning machine. When something fails, you can identify whether the issue is attention, encoding, organization, retrieval, scheduling, or application.

It should not feel like randomly trying study hacks.
