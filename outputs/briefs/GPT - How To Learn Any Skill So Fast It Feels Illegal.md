# GPT - How To Learn Any Skill So Fast It Feels Illegal

## Core Thesis

The source argues that fast skill learning is not produced by theory overload. It comes from quickly identifying the skill's practical structure, practicing early, getting feedback, and learning just enough theory to improve execution.

The learner should avoid the trap of preparing forever. Skill learning becomes faster when theory is subordinated to use.

## Compressed Takeaways

1. **Skill learning is execution-centered.** The learner has to do the thing.
2. **Theory is useful only when it improves practice.** More conceptual input is not automatically better.
3. **Early attempts reveal the real bottleneck.** Practice shows which subskills matter.
4. **Feedback beats passive exposure.** The learner needs information about performance, not just more instruction.
5. **The fastest path is targeted.** Identify the highest-leverage subskill and work it deliberately.
6. **Learning how to learn is a multiplier.** Skill acquisition improves when the learner can diagnose the process.

## Practical Workflow

1. Define the target skill in observable terms.
2. Identify the smallest useful version of the skill.
3. Try it earlier than feels comfortable.
4. Notice the failure mode.
5. Learn only the theory needed to fix that failure.
6. Practice again with better constraints.
7. Repeat with tighter feedback.

This connects to [[wiki/Techniques/Kolbs Experiential Cycle|Kolbs Experiential Cycle]]. Concrete experience exposes the gap; reflection identifies the cause; conceptualization proposes a change; experimentation tests it.

## What This Should Feel Like

It should feel like controlled exposure to failure. The learner should not feel fully ready before starting. The point is to generate real feedback.

It should not feel like collecting tutorials, building giant notes, or waiting until the theory is complete.
