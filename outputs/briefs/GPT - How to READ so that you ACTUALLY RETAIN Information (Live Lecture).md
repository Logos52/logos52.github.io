# GPT - How to READ so that you ACTUALLY RETAIN Information (Live Lecture)

## Core Thesis

The source argues that reading for retention requires active structure-building during reading, not passive exposure followed by later review. The learner should read with questions, organize information into relationships, and force retrieval while the material is still being processed.

## Compressed Takeaways

1. **Retention starts during reading.** Waiting until later creates rework.
2. **Reading should be question-led.** Questions give information a purpose.
3. **Structure beats volume.** A smaller amount of organized knowledge is more useful than a large pile of notes.
4. **Connection drives memory.** New ideas stick when attached to existing schemas.
5. **Retrieval should appear early.** The learner should test what can be reconstructed.
6. **Review should repair gaps.** Rereading should be targeted, not default.

## Practical Reading Loop

1. Ask what the source is trying to explain.
2. Predict the structure before reading deeply.
3. Read a small section.
4. Pause and reconstruct the point.
5. Connect it to the map.
6. Mark gaps or uncertainty.
7. Continue only when the section has been digested.

This source is highly compatible with [[wiki/Techniques/Bear Hunter System|Bear Hunter System]].

## What This Should Feel Like

It should feel like conversation with the text. The reader should be asking, predicting, checking, and reorganizing.

It should not feel like flowing through pages while hoping memory appears later.
