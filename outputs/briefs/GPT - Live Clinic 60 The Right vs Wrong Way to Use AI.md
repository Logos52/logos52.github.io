# GPT - Live Clinic 60: The Right vs Wrong Way to Use AI

## Core Thesis

AI helps learning when it sharpens the learner's questions, exposes missing information, and accelerates search around a thought process the learner is already doing. It damages learning when it performs the high-order processing that the learner needed to struggle through: grouping, evaluating importance, building analogies, forming schemas, deciding relationships, and explaining why something matters. The danger is that AI can produce an artifact that looks like learning while the brain skipped the lift. The useful standard is simple: use AI like a strict mentor or advanced search engine, not like a replacement mind.

## Key Takeaways

- Use AI after forming a partial model, not before the first serious attempt.
- Ask from the gap: name what does not click, then ask for the missing information that would help you decide.
- Keep ownership of grouping, importance, analogy, relationship, and final judgment.
- Treat AI as a strict mentor: low-quality questions should feel embarrassing; high-quality questions should reveal prior thinking.
- Use AI for keyword collection, hypothesis checks, gap scanning after retrieval, practice questions, and procedural review.
- Avoid using AI to create the schema, rank importance, generate analogies for unfamiliar material, or replace Kolb-style self-feedback.
- The practical test is whether AI made you think better or merely gave you something easier to consume.

## The Learning Problem AI Can Distort

Modern learning already creates a bad cognitive environment:

- low context,
- high density,
- weak immediate relevance,
- too much information too quickly,
- too little apprenticeship-style feedback.

Learning techniques repair this by increasing context and controlling density. They make the brain process information in a way it can actually organize and remember. AI enters this environment as a powerful tool, but its power makes the shortcut problem worse. It can turn a difficult high-order task into a low-order consumption task.

The visible output improves. The learner may not.

## The Main Failure Mode

```text
complicated source
-> learner feels overload
-> AI summarizes, groups, explains, ranks, analogizes
-> learner reads the polished output
-> task feels easier
-> schema-building was outsourced
-> knowledge remains shallow
```

The artifact looks like the thing that learning would have produced. A summary, grouping, explanation, analogy, or clean list can look like processed knowledge. The missing step is that the learner did not generate it, test it, revise it, or decide what it means.

That means the learner usually moves into memorizing the AI's output. A high-order problem has been converted into a lower-order task.

## Good Use: Ask From The Gap

Good AI use starts from a felt uncertainty inside your own thinking.

```text
I have a partial structure
-> something does not click
-> I identify the missing information
-> I ask AI for that information
-> I use the answer to make my own judgment
```

The question should preserve ownership. You are still deciding what matters, how things connect, and where the answer belongs.

Good questions:

- "I think A leads to B only when C is present. Is that relationship accurate?"
- "What information would help me decide whether these two concepts belong in the same group?"
- "What perspectives am I missing from this explanation?"
- "Generate practice questions that test whether I understand this relationship."
- "What assumptions does this answer depend on?"

Weak questions:

- "Explain this to me."
- "Why is this important?"
- "Group these concepts by importance."
- "Make an analogy so I understand it."
- "Tell me how this fits together."

The weak questions ask AI to create the schema. The good questions ask AI for information that helps you create the schema.

## The Strict Mentor Frame

Treat AI like a mentor who judges the quality of the question. A good mentor does not reward lazy questions. The standard is not politeness. The standard is whether the question proves that you have already tried to think.

The useful discipline:

1. Form an initial model yourself.
2. Notice where it breaks.
3. Ask for the specific missing information.
4. Use the answer to revise your model.
5. Keep final judgment in your own hands.

The learner should leave a session with better questions, sharper uncertainty, and a revised structure. If the session only produces easier text to consume, the AI probably replaced the work.

## Valid Jobs For AI

### Keyword Collection

Collecting keywords is necessary but low-value. AI can generate the initial field of terms so the learner can begin organizing. The learning happens after the list appears: comparing, grouping, testing relevance, and building the map.

### Hypothesis Checking

AI can test a proposed relationship once the learner has already formed it. This is especially useful when the question is too specific for ordinary search.

Example:

```text
I think A produces B only when C and D are both present.
Is that accurate, or are C and D interchangeable conditions?
```

The key is that the learner created the candidate structure first.

### Gap Scanning After Retrieval

Brain dump first. Ask AI to scan afterward.

This preserves the value of retrieval while using AI to catch blind spots. The benefit is not that AI creates the answer. The benefit is that it points to missing pieces after the learner has already generated the answer.

### Practice Questions

AI is strong for lower- and mid-level practice questions, and useful for creating variations of higher-order questions. Creating some questions manually is still valuable because question design itself trains judgment, but AI can remove a lot of low-value production load.

### Procedural Review

AI can help with error checking, code review, exemplar analysis, and extracting someone else's approach from a finished product. This works best when the learner still has to perform the skill and can use AI as feedback, not as substitute execution.

## Dangerous Jobs For AI

### Ranking Importance

Importance is relational. Asking AI why something matters usually asks it to build the schema for you. It decides the relationships, evaluates them, and gives you a conclusion. That shuts off the exact thinking you needed to train.

Better question:

```text
I think this concept matters because it changes this other concept.
What information would confirm or weaken that relationship?
```

### Creating Analogies For New Knowledge

Analogy generation is risky before mastery. If you already understand the topic, you can usually create the analogy yourself. If you do not understand the topic, you cannot reliably judge whether the analogy is correct. A clean analogy can make wrong understanding feel intuitive.

### Summarizing What You Need To Understand Deeply

Summaries can reduce friction, but they can also remove the processing that creates expertise. Use them for orientation, not as the main learning act.

### Kolb Feedback

Kolb reflections require nuance, self-feedback, and contextual judgment. Outsourcing them weakens a skill that should be trained internally.

## What It Should Feel Like

Good AI use feels active and slightly demanding. You are not relaxing into the answer. You are interrogating your own confusion.

Good signs:

- you ask fewer but sharper questions;
- the answer creates a next thinking step;
- your own model changes;
- uncertainty becomes more specific;
- you feel responsible for the final structure;
- you can explain why you accept or reject the answer.

Warning signs:

- the AI output feels instantly satisfying;
- you mostly read and nod;
- you ask for importance, grouping, summary, or analogy before trying;
- you cannot tell whether the answer is correct;
- the answer becomes something to memorize;
- your own schema is still vague afterward.

## Implications For This Knowledge Base

This is directly relevant to [[wiki/Concepts/Knowledge Base as Thinking Partner|Knowledge Base as Thinking Partner]]. The knowledge base should not become a machine that produces dopamine from new connections while the first brain stays passive. It should act as a prompt field: a place to generate better questions, expose gaps, and point back into active use.

The safest workflow:

```text
read source
-> attempt own model
-> ask AI for missing information or blind spots
-> revise model
-> test with retrieval or application
-> file only what changes future action
```

## Questions To Use In Practice

- What part of this am I trying to avoid thinking through?
- Did I create a model before asking?
- Am I asking for the answer, or for information that helps me decide?
- What judgment remains mine after AI responds?
- Would this prompt impress a strict mentor?
- What should I retrieve, explain, or apply after this answer?
