# How I Use It

This page contains practical notes on how the MG & Kolbs system is currently used (as of 2025).

### 1. Calendar Integration
I connect my **Tasks** database to the calendar so every task has a start and end time. This also helps generate proper rest blocks.

### 2. Auto-created Pages
Several pages (especially in Kolbs and Goal Tracking) are created automatically through templates and buttons. You can disable auto-creation if you prefer, but I recommend keeping the weekly evaluation + goal tracking automation.

### 3. Kolbs Cycling Button
There is a button in the **Kolbs** database that marks the current reflection as complete and automatically creates + opens the next one. If you use a numbering system, you will need to update the number manually after the new page is created.

### 4. Goals Database
The **Goals** database is currently the least optimised part of the system. It works, but the structure is still being refined.

### 5. Database Naming
When you first duplicate this template, many database names and relations will have a `(1)` at the end. Clean these up so relations work properly.

### 6. Filler Pages
I have included many example pages as templates. Feel free to delete anything that doesn’t serve you.
